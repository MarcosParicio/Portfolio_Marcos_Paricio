Procesando imágenes desde ficheros¶

Este dataset de imgenes es grande, con imgenes de mayor resolucin que las que hemos utilizado en el tutorial del MNIST, y cada una de ellas teniendo diferentes tamaos y relacin de aspecto. Adems, mientras que para el MNIST tenamos una funcin de keras que preparaba los datos para nosotros, en esta ocasin tendremos que realizar el trabajo de carga y procesamiento de las imgenes.

Una forma conveniente de hacer todo este trabajo es a travs de la funcin Keras image_dataset_from_directory. Esta funcin crea un objeto Dataset de TensorFlow con todas las imgenes de un directorio, cargndolas en memoria de forma dinmica solo cuando la red neuronal necesita utilizarlas. Esta funcin tambin nos permite especificar algunas opciones de preprocesamiento muy tiles.

Por ejemplo, podemos crear un Dataset con los datos en la carpeta de training:

In [ ]:
from tensorflow.keras.preprocessing import image_dataset_from_directory

image_size = 256 # Se considera 32 como un tamao demasiado pequeo, se determina razonable 128 y 256 x 256 quiz el lmite ptimo
batch_size = 128 # Se considera 64 aumentable. Valores pares razonables de lote: 8, 16, 32, 64, 128, 256

train_dataset = image_dataset_from_directory(
    TRAINDIR, 
    image_size = (image_size, image_size),
    batch_size = batch_size, 
    label_mode = 'categorical'
)

Found 6082 files belonging to 6 classes.

Observa los parmetros que se han utilizado para configurar el dataset:

  • El directorio desde el que cargar las imgenes.
  • Un tamao de imagen (image_size) que se utilizar para redimensionar todas las imgenes cargadas a ese tamao comn, en este caso 32x32 pxeles.
  • El tamaos de los lotes (batch_size) de imgenes a ser generados. Ntese que definimos aqu este parametro en lugar de en el paso fit de la red, como hemos hecho en otros ejercicios, porque el objeto Dataset resultante har uso de esta informacin para mantener en memoria solo algunos batches de imgenes, ahorrando as memoria.
  • El modo de etiquetado (label_mode), esto es, la codificacin de las etiquetas a utilizar. categorical significa que utilizaremos la ya conocida codificacin one-hot.

Un objeto Dataset funciona de manera muy similar a un generador de Python, lo que significa que podemos iterar sobre l para obtener batches de imgenes ya preprocesadas. La normalizacin realizada por la VGG16 consiste en invertir el orden de los canales de color (RGB -> BGR), y restar los valores medios sobre todo el dataset ImageNet para cada canal de color por separado. Afortunadamente, la funcin preprocess_input que hemos importado hace todo este trabajo por nosotros. Adems, podemos incrustar esta funcin como la primera capa de nuestra red, cumpliendo el papel de la capa Rescaling que utilizamos en el apartado anterior. Esto es posible gracias a la capa Lambda, que permite construir una capa Keras en base a cualquier funcin de Tensorflow. De modo que, vamos a comenzar nuestro diseo de red con esta capa.

In [ ]:
from tensorflow.keras.layers import Lambda

model = Sequential()
model.add(Lambda(preprocess_input, input_shape=(image_size, image_size, 3)))

Tras esto, podemos aadir toda la red VGG16 como si fuera una nueva capa, y nuestras propias capas despus de ella. A continuacin tenemos un ejemplo de esta clase de diseo, aunque es importante destacar que es un diseo muy sencillo que contiene algunos fallos; una red real para hacer transfer learning debera tener un diseo mejor.
